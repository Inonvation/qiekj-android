# LightLife

将胖乖生活核心流程（饮水设备解锁 + 积分自动化）重构成原生 Android 应用，去除臃肿功能和广告，保留最实用的部分。安装包体积仅约 **2MB**。

---

## 技术栈

| 类别 | 技术 |
|------|------|
| 语言 | Kotlin |
| UI 框架 | Jetpack Compose + Material3 |
| 网络 | OkHttp + Retrofit |
| 序列化 | Moshi |
| 存储 | SharedPreferences + JSON 文件 |
| 构建 | Gradle + AGP 8.7.3 |

---

## 功能

### 饮水设备控制
- 一键解锁：选择历史设备，调用接口解锁，订单自动保存
- 今日统计：自动计算当日喝水次数与积分抵扣金额
- 订单记录：查看历史订单详情（原价、花券、积分抵扣）

### 自动化积分任务
- 签到 + 环境检查 + 浏览任务 + 任务列表
- APP 视频广告 × 20 次，支付宝视频广告 × 50 次
- 智能跳过：已执行的步骤自动跳过，中断后重跑不重复
- 暂停 / 继续 / 结束，实时进度条 + 执行日志
- 日志本地持久化，支持查看历史执行记录

### 界面与交互
- 底部导航：首页 / 积分任务 / 我的
- 主题支持：跟随系统 / 浅色 / 深色
- 触感反馈：按钮和开关操作附带振动
- Token 查看：一键复制当前登录凭据

### 数据与存储
- Token 自动保存，退出登录一键清除
- 积分统计：累计获得积分与抵扣金额持久化
- 订单历史：JSON 文件本地存储
- 任务状态：SharedPreferences 持久化，跨天自动归零

---

### 与原项目的区别

LightLife 基于 [wzs0512/qiekj-android](https://github.com/wzs0512/qiekj-android) 重构而来，主要改进：

**智能跳过**：签到、任务列表等步骤的完成状态通过本地记录保存，视频广告额外记录已完成次数。中断后重新执行时，已完成的步骤直接跳过，广告任务从上次的次数继续，不重复调用接口。

- 代码结构从单文件拆分为四个独立页面，职责更清晰
- 新增备份与恢复功能，支持导出 `.lif` 文件并在设备间迁移数据
- 执行日志持久化保存，可查看历史执行记录
- 新增设置页面，支持主题切换、触感反馈、日志管理等配置
- 构建产物新增 Release 包，启用 R8 优化压缩至约 2MB

## 使用方法

### 登录
1. 进入「我的」页面，输入手机号发送验证码
2. 输入验证码确认登录，Token 自动保存

### 解锁设备
1. 「首页」等待历史设备列表加载
2. 点击目标设备一键解锁，完成后自动记录订单

### 刷积分
1. 确保已登录，进入「积分任务」页面
2. 点击「开始执行自动化任务」
3. 可随时暂停 / 继续 / 结束任务
4. 任务中断后重新执行会智能跳过已完成的步骤

## 构建

### 环境要求
- Android Studio Hedgehog (2023.1.1) 或更高版本
- JDK 17
- Android SDK 35

### 构建步骤
```bash
# 克隆仓库
git clone https://github.com/your-username/light-life.git
cd light-life

# 使用 Android Studio 打开项目，或命令行构建
./gradlew assembleDebug      # Debug APK
./gradlew assembleRelease    # Release APK

# 使用构建脚本（自动管理版本号）
scripts\build.bat             # Windows
scripts\build.ps1             # PowerShell
```

### APK 归档
构建脚本会自动将 APK 归档到 `archive/` 目录，命名格式：`app-debug-v{版本号}.apk`
